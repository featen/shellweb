Single Thread

I am use to develop multi-threads applications in the past, there are lots of issues you need to take care if you want to deliver a stable and efficient program, sometimes it's fun, most of the time it's just painful. On the other hand, single thread applications are easy to write and less sync/mutex issue to worry about, it's much easier to write and maintain, a good example is redis.

In fact, we don't have to write multi-threads applications all the time, if you have a good design to seperate the work load, like placing a load balancer in front of your applications, you are free to write single thread programs.

Processor Affinity

Processor affinity, or CPU pinning, enables the binding and unbinding of a process or a thread to a central processing unit (CPU) or a range of CPUs, so that the process or thread will execute only on the designated CPU or CPUs rather than any CPU. https://en.wikipedia.org/wiki/Processor_affinity

An execute parameter is added for running aehttpd, -c <cpu> will bind the process to a cpu core, which is an extend step of Single Thread. This only works on Linux for now.
